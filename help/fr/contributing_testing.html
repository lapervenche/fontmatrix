Tests et rapports

Tests

Les tests contribuent à rendre les applications solides comme un roc. Comme nous sommes un projet communautaire, nous comptons sur vous, oui — vous, nos chers utilisateurs.

Nous ne vous encourageons pas vraiment à compiler le code source le plus instable et de pointe, mais si vous trouvez des bugs dans la dernière version publiée, n'hésitez pas à nous les signaler.

Plantages

Vous avez donc trouvé un moyen reproductible de faire planter Fontmatrix. Si vous êtes sous Linux, veuillez utiliser l'application appelée gdb pour créer un rapport que les programmeurs appellent backtrace. Voici comment procéder :

  1. Installez gdb via le gestionnaire de paquets
  2. Ouvrez un terminal
  3. $ gdb fontmatrix
  4. La console de gdb apparaît
  5. > run
  6. Fontmatrix démarre, un peu plus lentement que d'habitude
  7. Reproduisez le plantage
  8. Retournez au terminal
  9. > bt
  10. Copiez la sortie avec la souris et collez-la quelque part
  11. > quit

Si vous êtes sous Windows ou Mac ou si vous n'avez tout simplement pas le temps de vous embêter avec gdb, au moins assumez et dites-nous exactement ce que vous avez fait.

Signalement de bugs et demande de fonctionnalités

Vous pouvez soumettre des rapports de bugs à notre bugtracker (pas besoin de s'inscrire). Ne vous inquiétez pas du nom "Undertype", car Fontmatrix appartenait à l'origine à ce projet plus vaste.

Un bon rapport de bug utile contient :

  1. Liste des actions ayant conduit au bug
  2. Un backtrace, si l'application a planté (voir ci-dessus)
  3. Informations sur votre système

Les informations typiques sur votre système que nous apprécierions :

  • Linux : nom et version de la distribution, version de Qt, version de Fontmatrix
  • Mac OS X : version de Mac OS X, version de Fontmatrix
  • Windows : version de Windows, version de Fontmatrix

Si vous souhaitez demander une nouvelle fonctionnalité, utilisez ce même traqueur.
