Traduction de l'entrée de bureau

Sur les systèmes Linux, les fichiers .desktop sont utilisés pour construire les menus système qui listent les applications disponibles aux utilisateurs. Fontmatrix est également fourni avec un tel fichier. Il est situé dans le répertoire racine avec le code source et est installé dans /usr/share/applications ou /usr/local/share/applications, selon vos préférences.

Pour obtenir une entrée de menu localisée, vous devez faire une chose très simple :

  1. Ouvrez ce fichier dans votre éditeur de texte préféré et assurez-vous de l'ouvrir en tant que fichier texte encodé en UTF-8.
  2. Créez une nouvelle entrée qui ressemble à "Comment[LANG]=Un gestionnaire de polices", où LANG est un code à deux lettres pour votre langue (tel que référencé par ISO-639-1) et tout ce qui suit "=" est traduit.
  3. Créez une nouvelle entrée qui ressemble à "GenericName[LANG]=Un gestionnaire de polices".
  4. Sauvegardez.
  5. Testez en exécutant 'sudo make install' (si Fontmatrix est déjà installé ou simplement 'sudo cp fontmatrix.desktop /usr/share/applications/') et regardez dans le menu.

Envoyez-nous le fichier mis à jour.

Traduction de l'interface utilisateur

La toute première chose dont vous avez besoin pour traduire Fontmatrix dans votre langue maternelle est d'obtenir le code source de la branche de développement actuelle. Pour ce faire, vous avez besoin d'un client Subversion (svn étant le client en ligne de commande habituel). Ensuite, tapez

svn co http://svn.gna.org/svn/undertype/trunk/tools/typotek fontmatrix

quelque part dans votre répertoire personnel pour récupérer le code source, afin d'y avoir toujours accès.

Ensuite, vous devrez installer le paquet de développement Qt qui contient Linguist — l'application pour vous aider à traduire. Sur Linux, démarrez votre gestionnaire de paquets et cherchez un paquet nommé quelque chose comme qt4-dev, installez-le.

Les étapes suivantes sont les suivantes :

  1. Allez dans le répertoire de niveau supérieur du code source.
  2. Créez le répertoire de construction, par exemple : mkdir build et accédez-y.
  3. Exécutez la commande Cmake avec les options que vous souhaitez, plus -DTRANSLATOR:string=LOCALE, où LOCALE est un code à deux lettres pour votre langue (tel que référencé par ISO-639-1), en minuscules, par exemple fr pour le français ou de pour l'allemand. Si nécessaire, vous pouvez spécifier un code de pays et ainsi former une chaîne comme fr_FR
  4. Exécutez "make translation" pour créer un nouveau fichier de traduction dans le répertoire fontmatrix/src/messages/.
  5. Ouvrez votre fichier fontmatrix-LOCALE.ts dans Linguist et commencez à traduire.
  6. Pour tester votre traduction, exécutez make install dans le répertoire de construction. Cela créera un fichier binaire appelé fontmatrix-LOCALE.qm (et Fontmatrix lui-même si vous n'avez pas exécuté cette commande auparavant) et l'installera au bon endroit. Si le LOCALE pour lequel vous traduisez n'est pas celui qui est défini pour votre système, exécutez LC_ALL=LOCALE fontmatrix pour indiquer à Fontmatrix de charger ce LOCALE spécifique.
  7. Lorsque votre travail est terminé, compressez ce fichier fontmatrix-LOCALE.ts dans une archive ZIP, GZ ou BZ2 et soumettez-le via le suivi des correctifs et des patchs ou informez-nous via IRC et nous vous aiderons à résoudre ce problème.

Voici quelques conseils pour vous aider à améliorer la traduction.

Traduire Fontmatrix prend du temps, il est donc préférable de traduire les parties de l'interface utilisateur que vous utilisez la plupart du temps. Cela vous donnera un faux, mais utile sentiment d'accomplissement et de motivation pour terminer tout le travail.

Testez votre traduction aussi souvent que possible. C'est particulièrement important pour les dialogues que vous utilisez rarement.

Assurez-vous de trouver un bon équilibre entre des phrases et des mots courts et faciles à comprendre. La langue anglaise est connue pour avoir des mots relativement plus courts, donc dans la plupart des cas, votre traduction rendra l'interface utilisateur un peu plus grande. Mais si vous commencez à utiliser des abréviations ou des synonymes plus courts qui ne correspondent pas tout à fait au contexte, les utilisateurs n'apprécieront pas non plus.

Beaucoup de messages traduisibles utilisent des variables comme %1. Celles-ci sont remplacées par certaines valeurs. Par exemple, dans "Block (%1):" (onglet Glyphes), cette variable est substituée par le pourcentage de couverture d'un bloc Unicode choisi, par exemple : "Block (74%):". Lorsque vous tapez ces variables manuellement, vous pouvez faire une erreur et l'astuce avec une variable ne fonctionnera pas. Il est donc préférable de coller le texte original dans le champ de saisie de la traduction en appuyant sur Ctrl+B dans Linguist, puis de remplacer ce texte original par la traduction, en laissant toutes les variables présentes intactes.

Une bonne partie des messages traduisibles sont des noms de blocs Unicode et de fonctionnalités OpenType. Même si ces technologies sont destinées à servir une vaste communauté mondiale avec toutes ses langues et écritures complexes, il est courant d'avoir peu ou pas d'informations à leur sujet dans une langue autre que l'anglais. C'est le 21ème siècle pour vous. Veuillez donc vous référer à de bons dictionnaires solides pour les noms corrects des langues utilisées pour la plupart des blocs Unicode, à la spécification Unicode pour le reste des blocs et à la spécification OpenType pour les noms des fonctionnalités OpenType que vous voyez dans l'onglet Exemple de texte.

Traduction de la documentation

La documentation se trouve dans le répertoire help et se compose d'un fichier menu.xml qui définit la table des matières, de fichiers HTML avec du texte et des illustrations dans des fichiers PNG ou JPEG.

Chaque traduction est conservée dans son propre répertoire nommé avec un code de langue à deux lettres comme ru pour le russe ou es pour l'espagnol. Téléchargez donc le code source, décompressez-le et commencez par créer ce répertoire.

Copiez maintenant le contenu du répertoire en dans le répertoire de votre langue. Avant de commencer à traduire, ouvrez le fichier help/LANGUAGE/CMakeLists.txt dans votre éditeur préféré et remplacez toutes les instances de "_EN" par "_LANGUAGE", où LANGUAGE est également un code à deux lettres pour votre langue, en majuscules. Ensuite, ouvrez help/CMakeLists.txt et ajoutez une autre ligne qui ressemble à ceci :

ADD_SUBDIRECTORY(${FM_HELP_DIR}LANGUAGE)

où LANGUAGE est un code à deux lettres pour votre langue, cette fois en minuscules.

Ces deux fichiers sont utilisés par le système de construction pour savoir quels fichiers installer.

Commencez à traduire. Il est préférable de traduire d'abord la table des matières et de continuer avec le contenu réel plus tard. Pour traduire la table des matières, ouvrez le fichier menu.xml dans l'éditeur de votre choix et traduisez les valeurs de chaque attribut text. Par exemple, pour <area text="Setting up" file="setting_up.html"> le texte traduisible sera "Configuration".

Pour tester votre traduction, ouvrez une fenêtre de terminal, allez dans le répertoire de niveau supérieur avec le code source, reconstruisez et réinstallez Fontmatrix. Après l'avoir reconstruit pour la première fois, vous n'aurez qu'à exécuter 'sudo make install' pour mettre à jour les fichiers.

Si l'interface utilisateur n'est pas localisée, vous voudrez peut-être le faire avant de traduire la documentation. Certains utilisateurs pourraient se plaindre et vous dire que l'interface utilisateur en anglais est une norme de facto et que la documentation localisée ne devrait donc faire référence qu'à l'interface utilisateur anglaise. Mais c'est juste parce qu'ils ont grandi en utilisant des logiciels non localisés, alors ne vous inquiétez pas, laissez-les faire les malins et continuez.

Certaines illustrations affichent en fait d'autres applications comme FontForge qui peuvent ne pas être localisées pour votre langue. Dans ce cas, utilisez simplement les captures d'écran déjà disponibles de la version anglaise de la documentation.

Les traductions anglaise (et russe) ont des captures d'écran avec le thème Dust pour GTK+ et Metacity (et le moteur GTK+ pour Qt), et la police Droid Sans 9pt. Vous n'avez pas à essayer de reproduire cela, mais veuillez être visuellement cohérent dans toute votre traduction.

Veuillez conserver toutes vos illustrations en dessous de 800 pixels sur le côté le plus long.

Lorsque vous avez terminé, archivez le répertoire help/LANGUAGE avec votre traduction et envoyez-le-nous.
